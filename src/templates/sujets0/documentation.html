Génération d'exercices
Cette application permet de générer des exercices inspirés des Sujets 0 du ministère de l'Éducation nationale. Ces exercices et leurs corrections sont générés aléatoirement grâce à un moteur de calcul formel.
Structure des données générées

Les générateurs d'exercices produisent une structure de données uniforme via la fonction missive(). Cette structure contient les données suivantes :

Champ Description Format Présent dans
beacon Identifiant unique de l'exercice Chaîne de caractères Tous les générateurs
statement Énoncé textuel de l'exercice (format LaTeX simple) Chaîne de caractères Tous les générateurs
statement_html Version enrichie de l'énoncé avec mise en forme HTML Chaîne HTML Tous les générateurs
answer Objet contenant les différents formats de la réponse Objet JSON Tous les générateurs
components Variables utilisées pour générer l'exercice Objet JSON Tous les générateurs
mask Préfixe à ajouter à la réponse (ex: "F=") Chaîne de caractères Certains générateurs (ex: 02)

L'objet answer contient les champs suivants :

Champ Description
latex Représentation LaTeX de la réponse
simplified_latex Représentation LaTeX simplifiée (souvent préférée pour l'affichage)
sympy_exp_data Représentation interne de l'expression pour le calcul formel
formal_repr Représentation formelle de l'objet mathématique (format Python)

Affichage des résultats

Le rendu des exercices dans l'interface suit cette logique :

  • Si statement_html existe, il est utilisé prioritairement pour un affichage enrichi
  • Sinon, statement est utilisé avec un formatage minimal
  • Pour les réponses, simplified_latex est préféré pour l'affichage
  • Le rendu LaTeX est effectué par KaTeX pour les formules mathématiques
Flux de données et traitement

Voici le flux de données complet du système de génération :

  1. Génération côté Python : Les générateurs produisent des données via missive()
    components → solve → question → missive
  2. Exécution via Nagini : executeGeneratorWithSeed dans index-nagini.js exécute le code Python et traite le résultat
    Code Python → Exécution → Extraction du résultat JSON
  3. Traitement des résultats : executeAllGenerators dans index-generators.js orchestre l'exécution
    Validation du formulaire → Sélection des générateurs → Exécution → Stockage des résultats
  4. Affichage : displayStudentResults dans index-results.js construit l'interface
    Préparation HTML → Analyse des résultats → Construction des cartes → Rendu LaTeX

Note technique : Le traitement des données dans index-results.js gère plusieurs cas d'affichage et implémente une logique robuste pour traiter toutes les variations possibles des données renvoyées par les générateurs.

Gestion des erreurs

Le système traite plusieurs cas d'erreur possibles :

  • Questions manquantes ou mal formées
  • Générateurs défaillants (erreur Python)
  • Données incomplètes ou incorrectes
  • Problèmes de rendu LaTeX

Extension du système

Pour ajouter un nouveau type d'exercice :

  1. Créer un nouveau générateur Python suivant la structure des existants
  2. S'assurer que la structure de missive est cohérente
  3. Tester avec différentes valeurs de seed pour vérifier la robustesse
  4. Ajouter le générateur dans la liste des générateurs disponibles
